# Architecture TODO: migrate homework images to Firebase Storage

**Дата добавления:** 2026-05-05
**Статус:** Pending
**Приоритет:** Средний — не блокер для запуска, но обязательно до 5+ активных учеников.

---

## Проблема

Поле `imageUrls` в коллекциях `homework` и `hwSubmissions` содержит base64-encoded JPEG'и прямо в Firestore-документах.

- Один документ homework ≈ 60 KB
- 10 ДЗ → дашборд тащит 600 KB при каждой загрузке
- 50 ДЗ → неюзабельно
- Та же проблема в `hwSubmissions` (фото ответов студентов)

Корень: `where("userId","in",[uid,""])` возвращает весь список ДЗ сразу, включая base64 в каждом документе.

---

## Решение

1. **Подключить Firebase Storage** (если ещё не подключен).
2. **Изменить код загрузки изображений:**
   - `file → Storage upload → получить download URL → сохранить URL в Firestore`
   - Затронутые места: форма создания ДЗ (AdminScreen), форма сдачи ДЗ (DashboardScreen `handleHwSubmit`)
3. **Заменить формат поля:**
   - `imageUrls: [base64, ...]` → `imageUrls: ["https://firebasestorage.../...jpg", ...]`
4. **Ленивая загрузка изображений:**
   - В списке ДЗ показывать только текст/заголовок
   - Картинки загружать только при раскрытии/открытии конкретного ДЗ
5. **Storage Security Rules:**
   - Студент читает только файлы, загруженные для него (`/homework/{uid}/...`)
   - Студент загружает только в свою папку (`/submissions/{uid}/...`)
6. **Миграция существующих данных (опционально):**
   - Скрипт: читает base64 из Firestore → загружает в Storage → записывает URL обратно
   - Нужен только если текущие base64-данные имеют ценность
