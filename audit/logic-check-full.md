# Логик-аудит dailyTasks — решение задач и сверка с ключом

**Метод.** Для каждого вопроса под-агент независимо решает задачу по `question`+`options`
(не подсматривая `correct`/`explanation`), определяет свой индекс правильной опции, затем
сверяет с полем `correct` и проверяет, к тому ли индексу приходит `explanation`. Каждый банк
(40 задач) решается «на свежую голову» отдельным агентом. Помечаемые расхождения
перерешиваются для защиты от ложных срабатываний.

**Типы проблем:** `возможная ошибка correct` (моё решение ≠ `correct`),
`explanation противоречит correct` (вывод объяснения ≠ `correct`),
`непроверяемо` (нельзя однозначно судить: дубль опций / недоопределено / нужен рисунок).

### Детали ключевой находки — `factoring_common_out #8`

Задача: разложить $144m^5n^4k^3 - 108m^4n^5k^2 + 72m^3n^3k^4$.
НОД(144,108,72)=36; мин. степени $m^3, n^3, k^2$. Деление среднего члена:
$108m^4n^5k^2 / 36m^3n^3k^2 = 3mn^2$. Верное разложение —
$36m^3n^3k^2(4m^2nk - 3mn^2 + 2k^2)$. Все 4 опции содержат `-3n^2` (или `-3n^2k`) вместо
`-3mn^2` → точного ответа в опциях нет; `correct=0` указывает на задуманную (с верными НОД и
крайними членами), но её средний член содержит опечатку. **Требует ручной правки опций.**

### 🔴 СИСТЕМНАЯ НАХОДКА — маркеры саморедактирования в `explanation`

Во вскрытых ошибках `explanation` почти всегда выводит ПРАВИЛЬНЫЙ ответ, но содержит фразы-улики:
«Ой…», «Исправим», «поставим 1 для баланса», «Изменяю Г на 10», «опечатка», «пусть будет».
То есть генератор давал верный вывод, но **путал маппинг ответа на индекс опции** и записывал неверный `correct`.

Скан всех 12 399 объяснений на эти маркеры:
**1041 вопрос (8.4%) в 232 банках** содержит маркер саморедактирования — дешёвый триаж-сигнал «вероятна ошибка ключа».
Топ-банки: `spatial_foundations`(29), `net_operations`(28), `spatial_figure_drawing`(20), `grouping_viz`(17),
`poly_rational_roots`(17), `dependency_types`(16), `system_graphical_solution`(16), `de_fundamentals`(14),
`inv_trig_compositions`(13)… (геометрия в топе — там часть задач визуальные, маркер может значить иное).

> **Важно:** маркер ≠ гарантия ошибки (иногда `correct` всё же поправлен). Это сигнал риска для адресной проверки,
> а не точный счётчик. Но корреляция с реальными ошибками в батче 2 — высокая.

## Корневая причина (детерминированный баг генерации)

`explanation` почти всегда выводит ПРАВИЛЬНЫЙ ответ, но поле `correct` указывает на другой индекс.
Объяснения содержат прямые улики: «индекс 0 занят, перемещаем», «Сдвинуто на индекс 2»,
«поставим 1 для баланса», «Вариант В: 72 (Исправлено)». Это баг распределения правильного ответа
по индексам (видимо, чтобы не все ответы были на idx0) — он перезаписывал `correct` мимо истинного индекса.
