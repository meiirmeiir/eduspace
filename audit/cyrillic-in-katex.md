# Корневая причина: «текст вопроса как `?????`, а формулы/числа целы»

## Диагноз (доказан)

Кириллический текст помещён **внутрь LaTeX-математики `$...$`**, а приложение рендерит
`$...$` через **KaTeX** (react-katex). Математические шрифты KaTeX (`KaTeX_Main`,
`KaTeX_Math`, …) содержат **только** латиницу, греческие буквы, цифры и матсимволы —
**кириллических глифов в них нет**.

- Цифры/операторы/`^2`/`\pi` берутся из `KaTeX_Main` (глифы есть) → **рендерятся корректно**.
- Кириллица (`см`, `кольца`, `НОК`) в `KaTeX_Main` = `.notdef`. KaTeX навешивает класс
  `cyrillic_fallback` со шрифтом `KaTeX_Main, "Times New Roman", serif` и надеется на
  **системный fallback**. Там, где «Times New Roman»/`serif` с кириллицей доступны
  (Windows-десктоп, мой Chromium) — буквы видны (засечным начертанием). Там, где такого
  fallback нет (часть Android/Linux/мобильных) — глиф не находится → **`?????`**.

### Эмпирические доказательства (live DOM)
- В DOM текст — **корректные кириллические кодпоинты** (U+0420 Р, U+0435 е…), реальных
  `?` (U+003F) нет → это **не порча данных** и **не битый UTF-8** (байты чисты, подтверждено).
- Узлы кириллицы внутри `.katex` имеют класс `mord cyrillic_fallback`,
  `font-family: KaTeX_Main, "Times New Roman", serif`.
- Canvas-проба: ширина «с» в `KaTeX_Main` == ширине в несуществующем шрифте (`.notdef`) →
  **`KaTeX_Main` физически не умеет кириллицу**; латиница/цифры — умеет.
- Шрифты приложения (Inter, Montserrat) кириллицу содержат и грузятся — **дашборд и
  обычный текст вопроса (MathText) рендерятся идеально**. Проблема строго в math-режиме.

### Что НЕ является причиной (проверено)
- Не шрифт `.question-text` (Montserrat с кириллицей, грузится).
- Не регэкспы: ни `MathText.jsx`, ни `LatexText.jsx`, ни `generateQuestion` кириллицу не
  трогают (LatexText стрипает только CJK — `一–鿿` и пр., не кириллицу).
- Не данные: байты чисты, в DOM настоящая кириллица.

## Варианты исправления

### A. Глобальный CSS-фикс (рекомендую как немедленный — 1 правило, без риска данных)
Добавить кириллице в KaTeX гарантированный fallback на **уже загруженный** веб-шрифт с
кириллицей (Inter), чтобы рендер не зависел от наличия «Times New Roman» в системе:

```css
.katex .cyrillic_fallback { font-family: KaTeX_Main, 'Inter', 'Times New Roman', serif; }
```

Латиница/цифры/матсимволы по-прежнему берутся из `KaTeX_Main` (он первый в цепочке) —
меняется только источник для кириллических `.notdef`. Чинит все 194 поля и любые будущие
на всех платформах. Риск минимальный.

### B. Чистка данных (правильная типографика, опционально, поверх A)
Вынести кириллицу из math-режима в банках:
- `$см^2$` → `см²` (или `см$^2$` — степень в math, единица — текстом);
- `$S_{кольца}$` → `$S_{к}$` либо `$S$ кольца`;
- `$НОК(12,15)$` → `НОК$(12,15)$`.
Делать через backup + dry-run + подтверждение (правило записи в dailyTasks).

**Минимально-достаточно — A.** B — для чистоты вёрстки.
